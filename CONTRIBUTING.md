# Contributing

Thank you for your interest in improving RR-framework.

Because this repository contains a methodology, questionnaire model, scoring logic, and Excel workbook, contributions should be structured and traceable.

## What contributions are useful

- Corrections to questions, scoring criteria, or documentation.
- New industry templates.
- New crisis scenario mappings.
- Improvements to risk libraries and recommendation logic.
- Improvements to formulas or workbook usability.
- Additional examples and user guidance.
- Bug reports with workbook version and affected sheet names.

## Contribution rules

By contributing, you agree that your contribution may be included under the same repository license: CC BY-NC-SA 4.0 for methodology and documentation. If code is later added under a separate code license, code contributions may be governed by that license.

Please do not submit proprietary, confidential, client-specific, or personal data.

## How to submit changes

1. Fork the repository.
2. Create a branch with a clear name.
3. Make the proposed change.
4. Update documentation if the logic changes.
5. Open a pull request.
6. Describe the problem, proposed change, and expected impact.

## Workbook contribution checklist

Before submitting workbook changes:

- Confirm workbook opens in Microsoft Excel desktop.
- Confirm no formula errors such as `#REF!`, `#DIV/0!`, `#VALUE!`, or `#NAME?` appear in visible calculation outputs.
- Confirm new questions have stable Question IDs.
- Confirm new questions are mapped to domain, subdomain, standards, risk library, and recommendation logic where applicable.
- Confirm no confidential data is included.
- Confirm copyright and license notices remain intact.
